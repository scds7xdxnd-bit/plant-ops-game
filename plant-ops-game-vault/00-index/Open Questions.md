# Answered Questions

#### Game Identity

| Question | Answer |
|---|---|
| First target player | Primary: chemical engineering student. Secondary: junior process engineer or process design trainee. |
| Realism level | Simplified-realistic: real engineering categories, solvable in 5 to 10 minutes, no dense legal text or full plant calculations. |
| Senior engineer feedback tone | Firm, specific, design-review style. Not cute, not overly academic, not quiz-like. |

Decision: the game is a design-basis interpretation game for chemical engineering students and early process design trainees, where the player acts as a junior process engineer converting a simplified BoD or DBM into early process design decisions.

#### Domain Scope

| Question | Answer |
|---|---|
| First product | Keep Solvex-A as a fictional specialty solvent additive. |
| Plant type | Continuous/batch hybrid. |
| First PFD Builder Lite blocks | Feedstock Storage, Feed Preparation / Dosing, Reactor, Heat Exchanger / Cooling System, Separator, Product Tank, Environmental Treatment. |

#### Gameplay Scope

| Question | Answer |
|---|---|
| Decision format | Use checkboxes/cards for selection, then drag-and-drop for workflow placement or PFD ordering. |
| Missing data in hard mode | Yes. Hard mode must allow missing data or assumption required as a valid answer. |
| Over-selection penalty | Light in Easy, moderate in Medium, strict in Hard. |
| Explanation timing | Easy: immediate. Medium: brief hint immediately, full explanation after submission. Hard: review feedback only after submission. |

#### Production Scope

| Question | Answer |
|---|---|
| First target | Browser prototype. Paper may test logic, but the first product target is playable in browser. |
| Classroom support | Yes, lightly: instructor notes, printable scenario sheets, and review screens. No teacher dashboard yet. |
| Scenario editing | Start with YAML/JSON, add no-code editor later. |
| FEA and 3D modeling | Explicitly out of MVP. Use only 2D block diagrams or simple plant maps as visual rewards. |
