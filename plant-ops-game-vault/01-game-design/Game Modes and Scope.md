# Game Modes and Scope

#### Purpose

This note defines what the first version should and should not try to be.

#### Recommended First Mode

The first playable version should be a single-player design basis exercise. The player reads a simplified BoD or DBM and maps it to process engineering decisions.

#### Candidate Game Modes

| Mode | Description | First-Version Fit |
|---|---|---|
| Design Basis mode | Player maps BoD or DBM clues to process decisions | High |
| PFD Builder Lite | Player selects and orders simplified process blocks | High |
| Equipment Datasheet Choice | Player chooses early datasheet assumptions | Medium |
| House of Quality mode | Player maps customer feedback to engineering qualities | Superseded |
| Scenario mode | Player handles one realistic customer-to-plant case | Later |
| Campaign mode | Multiple plants, customers, and strategic quarters | Later |
| Sandbox mode | Player edits plant data and creates experiments | Much later |
| Classroom mode | Instructor runs cases with students | Possible after prototype |
| Consulting workshop mode | Real company scenarios are modeled | Later, requires strong data controls |

#### First Vertical Slice

The first vertical slice should include:

- One simplified Solvex-A BoD or DBM
- Five decision categories
- Easy mode level flow
- Checkbox/card selection for process decisions
- Gantt-style level map
- Submit action
- Score and senior engineer explanation
- Simple unlock progression to the next design stage

#### Out Of Scope For First Prototype

- Multiplayer negotiation
- Real-time plant control
- Detailed physics or process simulation
- Large asset pipelines
- Procedural generation of complete factories
- Full ERP, MES, or supply-chain simulation
- Capex, downtime, and plant modification decisions
- FEA
- 3D modeling as a player task
- Full P&ID drafting

#### Design Question

How playful should the senior engineer feedback and mission framing be without making the chemical engineering feel fake?
